# Security Policy

## Reporting Security Vulnerabilities

Thank you for your interest in TRAE's security. We take security issues seriously.

### Bug Bounty Program

TRAE operates a **private bug bounty program on HackerOne**.

**How to participate:**

1. If you've discovered a security vulnerability or bug, please do not create a public GitHub issue
2. Instead, email us with your **HackerOne user ID** to request an invitation to our private program
3. Once invited, you can submit your findings through HackerOne

### What to Report

We're interested in:
- Security vulnerabilities
- Authentication/authorization issues
- Data exposure or privacy concerns
- Code execution vulnerabilities
- Other security-related bugs
